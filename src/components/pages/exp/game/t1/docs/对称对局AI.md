# 对称对局 AI 机制
AI（敌方单位）使用基于权重的搜索算法来决定每回合的最优出牌策略。

在游戏中不启用。

## 1. 决策流程
1.  **子集搜索**: 遍历手牌中所有可能的组合（1-5 张）。
2.  **牌型识别**: 识别每个组合的有效牌型和倍率。
3.  **增益评估**: 根据当前战斗状态（攻击/防御、血量百分比、敌方进攻强度）评估同花 Buff 的价值。
4.  **权重评分**: 对每个组合进行综合评分，选择得分最高的组合。

## 2. 评分权重表
| 指标 | 基础权重 | 备注 |
| :--- | :--- | :--- |
| **基础倍率** | 1.0 | 牌型自身提供的攻击/防御力。 |
| **真实伤害** | 2.0 | 穿透防御的高价值伤害。 |
| **中毒** | 1.0 | 持续性的无视护盾伤害。 |
| **护盾 (攻击时)** | 0.8 | 进攻时的防御倾向较低。 |
| **护盾 (防御时)** | 1.0 | 防御阶段的核心生存指标。 |
| **减伤** | 1.2 | 基于 `预计受伤害 * 减伤比例` 计算，评估价值略高于同值护盾。 |
| **回复 (HP < 40%)** | 2.0 | 低血量时的紧急救命优先级最高。 |
| **回复 (HP >= 40%)**| 0.5 | 健康状态下回复优先级降低。 |
| **愤怒 (下回合加成)**| 0.8 | 为后续进攻做铺垫。 |
| **净化** | 0.5 | 消除负面状态。 |

## 3. 阶段策略倾向
- **首轮进攻**: AI 会**优先尝试打出高级牌型**（如顺子、同花等）以触发“连击”，即使该组合的即时伤害略低于某些基础牌型。
- **追击/连击**: 受限于规则（禁止单张，<=3 张），AI 会选择得分最高的合规组合。
- **防御阶段**: AI 优先最大化“生存期望”，即 `防御力 + 护盾 + 减伤收益`。

## 4. 难度差异化设计
| 难度等级 | 搜索空间   | 牌型认知     | Buff 估值         | 策略特性                          |
|:---------|:-----------|:-------------|:------------------|:----------------------------------|
| **简单** | 随机 10 组 | 仅限基础牌型 | 忽略 (权重 0)     | 倾向于出单张，无连击意识。        |
| **普通** | 随机 50 组 | 全部牌型     | 线性 (固定权重 1) | 无动态权重调整，偶尔选择次优解。  |
| **困难** | 全量遍历   | 全部牌型     | 动态权重          | 连击诱导策略，精准计算生存/输出。 |

## 5. 致命缺陷
算法复杂度 O(2^n)，没有剪枝。
